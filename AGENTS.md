# AGENTS.md — NVIDIA Integrate Models Monitor

## Workflow Rules

- Sync before work: `git fetch --all --prune && git pull origin $(git branch --show-current) --rebase`
- Commit via `git-commit-push` skill only when explicitly requested
- Python runtime: 3.10 (pinned in `.github/workflows/monitor.yml`)
- Never alter `models.json` or `CHANGELOG.json` manually — these are auto-generated by `monitor.py`
- All changes to `monitor.py` must preserve the GitHub Actions `[skip ci]` commit pattern
- Dashboard files (`dashboard/`) are static HTML/JS/CSS — no build step required

## Context Loading Protocol

- At session start, always read `docs/CONTEXT-MAP.md` before exploring code
- Use it as the primary navigation index. Load only files referenced there
- Never load full directories or unrelated modules without explicit mapping
- Load `docs/SCOPE.md` and `docs/PLAN.md` at session initialization for requirement baseline and phased task alignment
- If combined file size exceeds 5KB, extract only the active phase, pending tasks, and requirement boundaries relevant to the current `docs/CHANGE-LOG.md` objective
- Validate implementation output against `docs/PLAN.md` checkbox status before marking `- [x]`
- Flag requirement deviations or scope drift in `docs/CHANGE-LOG.md` immediately; do not auto-modify upstream references

## Architecture & Token Control

- If `graphify-out/GRAPH_REPORT.md` exists:
  - Before reading source files, running grep, or generating code, read `graphify-out/GRAPH_REPORT.md`
  - Extract only the target modules, god nodes, and cross-dependencies relevant to the task
  - Retrieve files strictly by path listed in the report. Do not traverse directories recursively
  - After code changes, execute `graphify update .` before committing

## Token Budgets

- Session budget: 300K tokens
- Context-map + scope + plan: load once per session (< 5KB target)
- Source file reads: target < 10 files per task; use `docs/CONTEXT-MAP.md` to narrow scope
- Combined memory footprint of all `docs/` files: < 50KB

## AI Directives

- Language: Python 3.10 (monitor.py), JavaScript (dashboard/app.js), HTML/CSS (dashboard/)
- No database — state is file-based (`models.json`, `CHANGELOG.json`, `MODELS.md`)
- External API: `https://integrate.api.nvidia.com/v1/models` (read-only GET)
- Notification: Discord webhook via `DISCORD_WEBHOOK_URL` secret
- CI/CD: GitHub Actions (`monitor.yml`) — cron `17 1 * * *` UTC + `workflow_dispatch`
- Deployment targets: GitHub Pages (gh-pages branch) + Cloudflare Workers
- Secrets: `DISCORD_WEBHOOK_URL`, `CLOUDFLARE_API_TOKEN`, `CLOUDFLARE_ACCOUNT_ID`
- Never hardcode secrets or expose webhook URLs in commits
- Follow existing code style in each file — no formatting rewrites
